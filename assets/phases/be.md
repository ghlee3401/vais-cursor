### ⚙️ be — 백엔드 구현

1. **피처 레지스트리 참조** (`.vais/features/{feature}.json`) — 구현해야 할 기능 목록(`features[]`) 전체 확인. 각 기능 구현 후 `status`를 `"진행중"` → `"완료"`로 업데이트
2. **기획서의 코딩 규칙 참조** (`docs/02-plan/{feature}.md`)
3. **DB 설계 문서 참조** (`docs/05-design/{feature}-db.md` — 있는 경우)
4. 기획서의 데이터 요구사항 분석
5. **API 구현**:
   - 엔드포인트 구현 (기획서 API 컨벤션 준수)
   - 인증/인가 전략 적용
   - 에러 핸들링 (기획서 에러 처리 패턴 준수)
6. **DB 구현** (설계 문서가 있는 경우):
   - 마이그레이션 실행
   - ORM/쿼리 레이어 구현
7. 유효성 검증, 트랜잭션, 에러 핸들링 포함
8. 실제 코드 생성
